## Hallazgos nuevos de esta sesión (todos cerrados)

### Código / build

1. **`webassets.go` raíz duplicado** — paquete `webassets` en la raíz del módulo con `//go:embed all:web/build`, sin ningún importador (el real es `webassets/webassets.go` con `all:build`). Rompía `go build ./...` en un clone limpio sin build web previo. **Eliminado.**
2. **Makefile: fallback de pnpm roto** — `~/.nvm/versions/node/22/bin/pnpm` (recortaba `v22.14.0` → `22`; nvm usa el nombre completo con `v`). **Corregido** a `$(HOME)/.nvm/versions/node/$(node -v)/bin/pnpm`.
3. **install.sh: `ensure_go_local` ignoraba el `go` de PATH** — el comentario prometía "skip on a version match anywhere" pero el código sólo miraba `~/.local/go/bin/go` → descargaba ~70MB de toolchain aunque el sistema ya tuviera la versión correcta. **Corregido** (chequea `command -v go` primero; smoke-testeado el short-circuit).
4. **install.sh: leak de tempdir** — el `trap ... EXIT` de `ensure_go_local` era sobrescrito por el trap posterior del main → `$go_tmp` nunca se borraba (y referenciaba `$tmp` aún sin definir). **Corregido** con cleanup explícito en los tres paths.
5. **Mojibake en `mcp-tools update --help`** — `"…por defecto;保留 por compatibilidad"` (caracteres chinos en el help). **Corregido.**
6. **`sync.go`: hint de error apuntaba a `mcp-tools mcp-config`** (subcomando eliminado). **Corregido** → panel `/settings`.
7. **Directorio basura `~/` literal en la raíz del repo** — copia parcial de `plugins/mcp-tools-plugin` creada por algún `cp` con tilde sin expandir (ignorada por el gitignore global `*~`). **Eliminado.**

### Capa de instrucciones IA (skills / rules / agent docs)

Contexto: el refactor web-admin (PR #2) eliminó del CLI los subcomandos `up`, `logs`, `mcp-config`, `configure`, `<tool> install/upgrade` — pero 27 referencias en RULES.md, AGENTS.md, CLAUDE.md, 3 skills y docs/ADVANCED.md seguían instruyendo a los agentes a usarlos. Un agente que siguiera la escalación documentada fallaba con "unknown command". **Todo corregido** hacia los equivalentes reales: panel `/tools`·`/services`·`/settings`, `POST /api/tools/{key}/{verb}`, `POST /api/mcp-config/sync`, `docker compose -f ~/mcp-tools/dockers/compose.yaml --env-file ~/mcp-tools/.env …`.

8. **RULES.md**: escalación §0 (serena), §5 (logs), regla dura 8 (afirmaba que "mem0 levanta ollama+qdrant vía depends_on" — falso: `mem0-launcher` sólo exporta env y hace exec; no existe contenedor mem0). Firma del bug de mem0 actualizada: el error sano real es `ValueError: Top-level entity parameters frozenset({'user_id'}) are not supported in search()` (re-verificado en vivo 2026-07-13); pasar `filters={"user_id": ...}` como parámetro NO lo evita (el MCP inyecta el top-level igualmente).
9. **skills/mem0/SKILL.md**: contradicción interna — la tabla listaba `delete_memory`/`delete_all_memories` como tools pero el workflow de borrado decía "currently not exposed as a tool". Workflows de recall/save actualizados para no reintentar `search_memories` tras el `ValueError` conocido (ahorra una llamada rota por operación). Refs a `mcp-tools up/logs/mem0 install` corregidas.
10. **skills/serena/SKILL.md** y **skills/tokensave/SKILL.md**: refs a subcomandos eliminados corregidas. (Verificado en vivo que `find_declaration`/`find_implementations` SÍ existen en el server serena — la tabla de tools era correcta.)
11. **AGENTS.md + CLAUDE.md** (repo): mismas dos líneas stale corregidas en ambos.
12. **docs/ADVANCED.md**: refs de instalación por CLI → panel; imagen ollama documentada como `latest` cuando compose pinnea `0.31.1`; `internal/cli/up.go` citado como sync point cuando ya no existe (los callers reales de `OllamaComposeFiles` son `Ollama.Install` y `Ollama.Status`).
